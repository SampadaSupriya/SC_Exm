# Android App Security Testing – Notes
*(Static & Dynamic Analysis – Campus360 App)*

---

## 1. Android Studio – Create an Emulator (AVD)

1. Open Android Studio → **More Actions → Virtual Device Manager**
2. Click **Create a Virtual Device**
3. Select phone: **Pixel 6a**
4. Click **Next**
5. Select system image: **API 33 – "Tiramisu" – Android 13.0**
6. Click **Download**, then **Finish**

---

## 2. Setup ADB Environment Variable (Windows)

**Path to platform-tools:**
```
C:\Users\<your-username>\AppData\Local\Android\Sdk\platform-tools
```

**Steps:**
1. Open **File Explorer** → Right-click **This PC** → **Properties**
2. Go to **Advanced system settings**
3. Click **Environment Variables**
4. Under **System variables**, select **Path** → **Edit** → **New**
5. Paste the path:
```
%LOCALAPPDATA%\Android\Sdk\platform-tools
```
6. Click **OK** → **Finish**

**Verify setup (cmd):**
```
C:\> adb devices
```
Confirms the emulator/device is detected.

---

## 3. Rooting the AVD (rootAVD)

1. Go to **GitLab** and clone (or download ZIP) the `rootAVD` repo:
```
git clone https://gitlab.com/newbit/rootAVD.git
```
2. Extract all files (if downloaded as ZIP)
3. In cmd:
```
cd rootAVD
```
4. **List available AVD images:**
```
rootAVD.bat ListAllAVDs
```
→ shows the exact path to your AVD's `ramdisk.img` (or `ramdisk-qemu.img`)

5. **Root the running AVD:**
Run the root command against the correct `ramdisk.img` path found above.
*(This installs the Magisk app version inside the emulator.)*

6. **Reboot the emulator**

7. **Configure Magisk:**
- Open Magisk app → **Settings**
- Enable **Zygisk**
- Go to **Superuser** → **App and adb** → check the box → **Reboot**

8. **Verify root access (cmd):**
```
adb root
adb shell
su
# ← root access confirmed (prompt changes to #)
```

---

## 4. Installing & Testing the Target App

**Install APK via adb:**
```
C:\Users\<user>\Downloads> adb install campus360.apk
```

**Open Jadx-GUI** to review the decompiled source code of the app
(e.g. mount source from an F: drive / working folder)

---

## 5. JADX GUI + Sensitive Data Exposure in Logs

**Connect adb (example with explicit path):**
```
sudo ./mnt/c/Users/PG-ITISS/AppData/Local/Android/Sdk/platform-tools/adb.exe /bin/adb
adb shell
su
```

### ⚠️ Sensitive Data Exposure in Android Logs

**View full system logs:**
```
adb logcat # entire Android OS log stream
adb logcat -m <logs>
```

**Filter logs by app (pidcat):**
```
apt install pidcat -y
pidcat <app_package_name> # logs specific to that app can be seen
```

➡️ **Finding:** The **OTP** value was visible in the logs.

**Reproduction steps on Campus360 app:**
1. Open Campus360 app → **Forgot Password**
2. Enter phone number → OTP sent
3. Enter OTP → Verified
4. OTP value appears in `adb logcat` / `pidcat` output → **sensitive data exposure vulnerability**

---

## 6. Triggering an Activity Directly via ADB

```
adb shell su -c "am start -n com.campus360/com.campus360.activity.LoginActivity"
```
Used to launch a specific app activity directly (bypassing normal navigation) for testing.

---

## 7. Static & Dynamic Analysis – MobSF (Mobile Security Framework)

**Tool:** Google Play Store / GitHub – MobSF
Repo: `https://github.com/MobSF/Mobile-Security-Framework-MobSF`

**Setup via Docker:**
```
docker system prune -a -f # removes any previous images (cleanup)
docker pull opensecurity/mobile-security-framework-mobsf:latest
```

**Run the container:**
```
docker run -it --rm -p 8000:8000 -d opensecurity/mobile-security-framework-mobsf:latest
```
*Note: if a container of the same image is already running, it removes it and starts a new one.*

**Default login credentials:**
```
Username: mobsf
Password: mobsf
```

Access the dashboard at: `http://localhost:8000`
Upload the APK for **static analysis**, and connect a device/emulator for **dynamic analysis**.
